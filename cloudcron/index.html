CloudCron: Painless Cron Jobs in the Cloud

TL;DR: We're launching CloudCron (Beta) which allows you to run cron jobs in the cloud easily. Sign up with email, give us a Docker image name, command, arguments and a cron schedule and you're done. We'll run the command in our cloud infrastructure and send the output to you over either en email or a webhook endpoint.

At polyglot.network, we love open-source software. We're an OSS-only software development agency and our entire business is improving open-source software by adding features and bugfixes that our customers demand. There are some really nifty open-source tools out there which our customers want to run on a periodic basis. Here are a few examples:

  • curl, when run periodically, can act as a health-check.
  • LightHouse by Google can generate a detailed performance report of any webpage. This is the same tool that Chrome's has built-in as "Performance" tab of its Dev Console. Usage: lighthouse https://yoursite.example.com/
  • Apache Benchmark can load-test your webapp by firing concurrent requests.

In many of these cases, users want to receive the output of these commands to their email address or to a webhook endpoint, where this could be consumed programmatically. However, setting these up as periodic tasks is too cumbersome for many.

  • You need to provision a server. Setting up a billing account with AWS or GCP and ensuring that costs are contained - is not trivial.
  • Need to figure out a strategy for maintaining and rotating credentials
  • Install all the dependencies (such as an OS, libraries) and keep them up-to-date
  • Write custom code to send the output to an endpoint such as email or webhook

All this is not hard for developers, but it IS cumbersome and annoying. It should be dead-easy to run batch jobs in the cloud. Since polyglot.network is all about improving FOSS tools, we wanted to provide an easy way to do so. Enter CloudCron (Beta), which can run any Docker container at a specified schedule. Here's all you need to do:

  • Sign up using your email address. We use a magic-link for both registration and login, so there are no passwords to remember or rotate.
  • Tell us the name of the Docker image, the name of the command, and the arguments. As an example:
    • Docker image name: femtopixel/google-lighthouse
    • Command name: lighthouse
    • Arguments: https://yourwebsite.com --output=html --output-path=stdout --quiet --no-enable-error-reporting
  • Tell us the job frequency using the crontab syntax: 0 9 * * *
  • Specify the maximum number of times the job should be run. This puts an upper-bound on the cost.
  • And finally, specify the endpoint where you'd like to receive the results of each execution. The endpoint can be either an email or webhook.
  • We also have advanced options to configure the MIME/type and/or attachments, RAM size constraints etc.

This infrastructure is built on top of Google Cloud Components such as Cloud Scheduler, Cloud pub/sub, Cloud Functions, GKE, and CloudSQL to ensure reliability and scalability.

ANY command that's available as an image on Docker Hub, can be run. Bring your own ideas!
